Sistemas Operativos

Un sistema operativo (SO)

Es el software principal que gestiona el hardware y software de una computadora. Sirve como intermediario entre el usuario y los recursos de hardware, permitiendo que las aplicaciones funcionen correctamente. A continuación, se detalla todo lo fundamental sobre los sistemas operativos:

1. Definición y Funciones

Un sistema operativo es un conjunto de programas que:

  • Administra los recursos del hardware: Procesador, memoria, almacenamiento, dispositivos de entrada/salida.
  • Proporciona servicios básicos: Manejo de archivos, gestión de usuarios y permisos.
  • Permite la ejecución de aplicaciones: Sirve como plataforma para programas y aplicaciones.

Funciones principales:

  • Gestión de procesos:
    • Creación, ejecución y finalización de procesos.
    • Planificación y asignación de CPU.
    • Manejo de multitarea y multihilos.
  • Gestión de memoria:
    • Asignación y liberación de memoria.
    • Uso de memoria virtual.
    • Protección y organización del espacio de direcciones.
  • Gestión de dispositivos:
    • Control de hardware (impresoras, discos, teclados, etc.).
    • Comunicación con controladores de dispositivos.

2. Componentes de un Sistema Operativo

Los sistemas operativos se componen de varias partes esenciales:

  • Núcleo (Kernel):

    Es el corazón del sistema operativo. Administra los recursos del sistema y permite la comunicación entre hardware y software.

    • Monolítico: Todo el núcleo funciona como un único programa (ejemplo: Linux).
    • Microkernel: Dividido en módulos más pequeños y simplificados (ejemplo: Minix).
  • Gestor de procesos: Coordina la creación, eliminación y sincronización de procesos.
  • Gestor de memoria: Controla el acceso a la memoria principal y asigna espacios según las necesidades de los procesos.
  • Sistema de entrada/salida: Proporciona métodos para que los programas se comuniquen con dispositivos de hardware.
  • Sistema de archivos: Administra cómo se almacenan y recuperan los datos.

3. Clasificación de Sistemas Operativos

Se pueden clasificar según diferentes criterios:

Por su funcionalidad:

  • Monotarea: Solo ejecuta un proceso a la vez (ejemplo: MS-DOS).
  • Multitarea: Permite ejecutar varios procesos simultáneamente (ejemplo: Windows, Linux).

Por el número de usuarios:

  • Monousuario: Un solo usuario puede interactuar con el sistema a la vez.
  • Multiusuario: Permite que múltiples usuarios trabajen al mismo tiempo.

Por la arquitectura:

  • SO de escritorio: Diseñados para PCs personales (ejemplo: Windows, macOS).
  • SO de servidores: Optimizados para servidores y gestión de redes (ejemplo: Windows Server, Unix).
  • SO en tiempo real (RTOS): Procesan tareas con tiempos de respuesta garantizados (ejemplo: sistemas de control industrial).

4. Seguridad en Sistemas Operativos

Características esenciales:

  • Gestión de permisos y usuarios: Control de acceso mediante autenticación y permisos de archivos.
  • Encriptación: Proteger datos sensibles mediante cifrado.
  • Aislamiento de procesos: Evitar interferencias entre procesos en ejecución.
  • Parcheo y actualizaciones: Corregir vulnerabilidades mediante actualizaciones regulares.

Amenazas comunes:

  • Malware: Virus, gusanos, ransomware, etc.
  • Ataques de red: Suplantación de identidad, denegación de servicio (DoS), etc.
  • Errores de configuración: Permisos incorrectos o configuraciones inseguras.

5. Sistemas Operativos y la Virtualización

La virtualización permite ejecutar múltiples sistemas operativos en un solo hardware físico, gracias a tecnologías como:

  • Máquinas virtuales (VMs): Software como VMware, VirtualBox o Hyper-V.
  • Contenedores: Sistemas como Docker y Kubernetes que ofrecen entornos ligeros para aplicaciones.

Ventajas:

  • Optimización de recursos.
  • Reducción de costos operativos.
  • Mayor flexibilidad en el desarrollo y pruebas de software.
